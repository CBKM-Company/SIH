Why BlockChain ?

BlockChain gives power of Immutability, Transparency, Accountability, Security of transaction and assets.

In our project
We are using private blockchain network
As we cannot risk keeping the record in public blockchain because anyone can see the transaction . Although the data is Immutable but there will be privacy issues.

As BlockChain is Immutable therefore explicit tempering cannot be done also the data is cryptographically hashed , linked to each other and stored hence , it’s difficult to decrypt it .

How do things work ?

We are using Hyper-Ledger Fabric built by Linux foundation .

As you can see in the first fig (Hyperledger Infrastructure )
We have 4 docker containers

  1. CA Server: This container provides public and private keys for each user and has a track on the expiration period of these keys .MSP (Membership Service Provider ).

  2. Orderer : this keeps a track of sequence of transaction which is build on KAFKA (it helps in making real time streaming pipelines that are scalable )

  3. Peer : It is a node of a hyperledger network that has its own DataBase. We are using at least 4 peers for development but it can scale as per the requirement.

  4. ** Couch DB :** it is the Database used by the hyperledger to store unstructured data . And deep down It uses LevelDB.

  5. Client Peer is to provide RPC to client request

  6. Endorser peer : It checks the Certificates and details to validate the transaction.

  7. Anchor peer: it is used to receive updates and broadcasts these updates to the other peers in the organization. As Anchor peer is discoverable so we can communicate to other organizations by GOSSIP protocol.

  8. There are two more folder Config folder stores all the channel and peer configuration and Crypto folder stores all the crypto material like digital certificates .

As we mentioned that we will be providing the best security services but the question arises

How are we going to deliver the best security ?

We are storing millions of unstructured data mostly like text image pdfs …etc
Hence ,
We are Using asymmetric public-private key pairs provided by PKI (public key infrastructure ) from MSP (Membership service provider ) to encrypt-decrypt stored data. With RSA algorithm.

Just like SSL, we too are storing the data and enhancing the security by public-private key pairs.
And with the support of blockchain we double the level-up of the security as data is stored in encrypted form and linked to each other through hash which brings immutability.

How are we sharing the EHR?

We are providing Controlled Authority to patient

Hyperledger fabric provides secure transaction where two peer can communicate with each other without the knowledge of the other peers

So patient can initiate transaction through physical process like OTP or QR SCAN

And we verify it by private-public key pair just like SSL

We are also providing room for researchers where patients can send anonymous data for research purposes in labs and universities .
